```javascript
const BASE_URL = 'https://crm.andejiazheng.com/api';

const authenticatedRequest = (options) => {
  const token = wx.getStorageSync('access_token');
  
  wx.request({
    url: BASE_URL + options.url,
    method: options.method || 'GET',
    header: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json',
      'X-Client-Type': 'miniprogram',
      'X-Platform': 'wechat'
    }
  });
};
```

### 2. Service 层封装

<augment_code_snippet path="D:\安得家政\miniprogram\services\miniprogramCustomerService.js" mode="EXCERPT">
```javascript
class MiniprogramCustomerService {
  constructor() {
    this.baseUrl = '/customers/miniprogram';
  }

  async getCustomers(params = {}) {
    const response = await authenticatedRequest({
      url: `${this.baseUrl}/list?page=${params.page}&limit=${params.limit}`,
      method: 'GET'
    });
    return response.data;
  }

  async createCustomer(customerData) {
    const response = await authenticatedRequest({
      url: `${this.baseUrl}/create`,
      method: 'POST',
      data: customerData
    });
    return response.data;
  }
}
```
</augment_code_snippet>

## 🎯 核心特点

### 1. **前后端完全分离**
- ✅ 小程序只负责展示和交互
- ✅ 所有业务逻辑在后端处理
- ✅ 数据通过 RESTful API 交互

### 2. **独立的 Web 后台**
- ✅ 部署在独立服务器上（crm.andejiazheng.com）
- ✅ 可以同时服务小程序和 Web 管理端
- ✅ 统一的数据库和业务逻辑

### 3. **JWT 认证机制**
- ✅ 用户登录后获取 JWT Token
- ✅ 每次请求携带 Token
- ✅ Token 过期自动跳转登录页

### 4. **完整的 API 接口**
```
GET    /api/customers/miniprogram/list          # 获取客户列表
POST   /api/customers/miniprogram/create        # 创建客户
GET    /api/customers/miniprogram/:id           # 获取客户详情
PATCH  /api/customers/miniprogram/:id           # 更新客户
PATCH  /api/customers/miniprogram/:id/assign    # 分配客户
POST   /api/customers/miniprogram/:id/follow-ups # 创建跟进记录
GET    /api/customers/miniprogram/:id/follow-ups # 获取跟进记录
GET    /api/customers/miniprogram/statistics    # 获取统计信息
GET    /api/customers/miniprogram/employees/list # 获取员工列表
```

## 🚀 给"安得褓贝"的建议

### 方案对比

#### 方案 A：学习"安得家政"模式（独立服务器）

**优点：**
- ✅ 架构清晰，前后端完全分离
- ✅ 可以同时服务小程序和 Web 管理端
- ✅ 扩展性最强，可以接入其他平台
- ✅ 性能更好，不受云函数限制

**缺点：**
- ❌ 需要购买服务器（¥100-500/月）
- ❌ 需要配置域名和 SSL 证书
- ❌ 需要运维知识（部署、监控、备份）
- ❌ 开发周期较长（3-4个月）

**适合场景：**
- 团队有后端开发能力
- 预算充足（> ¥10万）
- 追求长期发展
- 需要接入多个平台

---

#### 方案 B：使用微信云开发（推荐⭐⭐⭐⭐⭐）

**优点：**
- ✅ 零成本快速上线
- ✅ 无需购买服务器
- ✅ 自动 HTTPS，无需配置
- ✅ 开发周期短（1-2个月）
- ✅ 维护成本低

**缺点：**
- ❌ 扩展性相对较弱
- ❌ 受微信云开发限制
- ❌ 云函数有冷启动问题

**适合场景：**
- 团队规模小（1-3人）
- 预算有限（< ¥5万）
- 需要快速上线
- 只服务微信小程序
