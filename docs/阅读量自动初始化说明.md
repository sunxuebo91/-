# 阅读量自动初始化功能说明 ✅

## 🎯 问题背景

**问题**：阅读量存储在**云数据库**（微信云开发），但文章内容存储在**MongoDB**（Node.js后端），这是两个独立的数据库！

**解决方案**：
1. 首次启动时，自动初始化云数据库中的阅读量
2. 加载文章列表时，从云数据库获取阅读量并合并到文章数据中

---

## ✨ 实现功能

### 1. 自动初始化（首次启动）

**触发时机**：小程序首次打开时

**工作流程**：
```
首页加载 → 检查是否已初始化 → 未初始化 → 调用云函数批量初始化 → 标记已初始化
                              ↓
                           已初始化 → 跳过
```

**初始化规则**：
- 给每篇文章设置随机初始阅读量（1000-5000）
- 只初始化一次，后续启动不会重复初始化
- 使用本地存储 `viewCountsInitialized` 标记

### 2. 阅读量合并（每次加载）

**工作流程**：
```
加载文章列表 → 从后端API获取文章内容 → 从云数据库获取阅读量 → 合并数据 → 显示
```

**数据来源**：
- **文章内容**：Node.js后端 MongoDB（标题、作者、封面图等）
- **阅读量**：微信云数据库（viewCount字段）

## 🚀 部署步骤

### 步骤 1：上传云函数

1. 在微信开发者工具中，右键点击 `cloudfunctions/articleService` 文件夹
2. 选择【上传并部署：云端安装依赖】
3. 等待上传完成

### 步骤 2：清除初始化标记（测试用）

在控制台运行：
```javascript
wx.removeStorageSync('viewCountsInitialized')
```

### 步骤 3：重新编译小程序

## 🔧 故障排查

### 问题 1：阅读量显示为 0

**原因**：云函数未上传或初始化失败

**解决**：
1. 检查云函数是否上传成功
2. 在控制台运行 `wx.removeStorageSync('viewCountsInitialized')`
3. 重新编译小程序

### 问题 2：控制台报错 "unknown action"

**原因**：云函数版本过旧

**解决**：重新上传云函数

### 问题 3：阅读量不更新

**原因**：点击文章时云函数调用失败

**解决**：查看控制台错误日志，检查云函数权限

---

## 📝 注意事项

1. **首次初始化只执行一次**，如需重新初始化，需清除本地存储标记
2. **阅读量存储在云数据库**，与后端MongoDB数据库独立
3. **每次加载文章列表都会从云数据库获取最新阅读量**
4. **点击文章时阅读量+1**，使用云函数原子操作保证准确性
