## 📱 如何使用测试页面

### 方法1：在微信开发者工具中直接访问

1. 打开微信开发者工具
2. 在编译模式中选择"添加编译模式"
3. 页面路径输入：`pages/test-user-api/index`
4. 点击"确定"并编译
5. 点击页面上的测试按钮

### 方法2：通过代码跳转

在任意页面的控制台中输入：

```javascript
wx.navigateTo({ url: '/pages/test-user-api/index' })
```

### 方法3：修改 app.json 首页（临时）

将 `app.json` 的第一个页面改为：

```json
{

    "pages/test-user-api/index",
    ...
  ]
}
```

## 📊 测试结果说明

### ✅ 成功（绿色）
- HTTP 状态码 200
- 响应数据中 `success: true`
- 接口正常工作

### ❌ 失败（红色）
- HTTP 状态码 200
- 响应数据中 `success: false`
- 接口返回业务错误

### ⚠️ 错误（黄色）
- HTTP 状态码 404/500
- 网络请求失败
- 接口不存在或服务器错误

### ⏳ 未测试（灰色）
- 尚未执行测试

## 🔍 常见测试结果

### 1. 接口不存在（404）

**日志输出**:
```
❌ 登录接口请求失败: request:fail
📡 HTTP 状态码: 404
```

**原因**: CRM 后端尚未实现该接口

**解决方案**: 
- 等待后端开发完成
- 或者使用 Mock 数据测试

---

### 2. 接口返回失败（业务错误）

**日志输出**:
```
⚠️ 登录接口返回失败: 用户不存在
📡 HTTP 状态码: 200
📦 响应数据: { "success": false, "message": "用户不存在" }
```

**原因**: 业务逻辑错误（如用户不存在、参数错误等）

**解决方案**: 
- 检查请求参数是否正确
- 检查后端业务逻辑

---

### 3. 接口调用成功

**日志输出**:
```
✅ 登录接口测试成功！
✅ 用户ID: 1
✅ 手机号: 13800138000
✅ 昵称: 测试用户
✅ 是否已授权手机号: 是
```

**说明**: 接口正常工作，可以继续集成到小程序中

## 💡 测试建议

### 1. 先测试登录接口
- 登录接口是基础接口
- 如果登录接口失败，注册接口也会失败

### 2. 检查 OpenID
- 确保能正常获取 OpenID
- OpenID 是所有接口的必需参数

### 3. 查看完整日志
- 日志中包含完整的请求和响应数据
- 可以帮助定位问题

### 4. 对比 Banner 接口
- Banner 接口是正常工作的
- 可以对比测试，确认网络环境正常
