## ✅ 已完成的工作

### 1. 复制核心文件

已从"安得家政"复制以下文件到"安得褓贝"：

- ✅ `miniprogram/utils/request.js` - HTTP 请求工具
- ✅ `miniprogram/services/auth.js` - 认证服务
- ✅ 修改 `miniprogram/pages/login/index.js` - 登录页面逻辑

### 2. 核心功能

**账号密码登录流程：**

```
用户输入账号密码
    ↓
调用 authService.login(username, password)
    ↓
POST https://crm.andejiazheng.com/api/auth/login
    ↓
返回 { success: true, data: { access_token, user: {...} } }
    ↓
保存到本地存储（access_token + userInfo）
    ↓
跳转到首页
```

**Token 验证流程：**

```
页面加载
    ↓
检查本地是否有 Token
    ↓ 有
调用 authService.validateToken()
    ↓
GET https://crm.andejiazheng.com/api/auth/me
    ↓ Token 有效
跳转到首页
```

## 🚀 如何使用

### 步骤 1：配置域名白名单

在微信小程序后台配置服务器域名：

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入"开发" → "开发管理" → "开发设置"
3. 在"服务器域名"中添加：
   - **request 合法域名**：`https://crm.andejiazheng.com`

### 步骤 2：开发环境测试

在开发者工具中：

1. 打开"详情" → "本地设置"
2. 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
3. 保存设置

1. 运行小程序
2. 进入登录页面
3. 点击"账号密码登录"按钮
4. 输入"安得家政"的账号和密码
5. 点击"登录"

## 📋 API 接口说明

### 1. 登录接口

**请求：**
```
POST https://crm.andejiazheng.com/api/auth/login
Content-Type: application/json

{
  "username": "zhangsan",
  "password": "123456"
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": "user_123",
      "name": "张三",
      "username": "zhangsan",
      "phone": "13800138000",
      "role": "admin",
      "permissions": ["resume:view", "resume:create"],
      "department": "销售部"
    },
    "openid": "oXXXXXXXXXXXXXXXXXXXXXXXXXXX"
  }
}
```

### 2. 验证 Token 接口

**请求：**
```
GET https://crm.andejiazheng.com/api/auth/me
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**响应：**
```json
{
  "success": true,
  "data": {
    "id": "user_123",
    "name": "张三",
    "username": "zhangsan",
    "role": "admin"
  }
}
```

## 🔐 本地存储说明

登录成功后，会在本地存储以下数据：

| Key | 说明 | 示例 |
|-----|------|------|
| `access_token` | JWT Token（主键） | `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` |
| `token` | JWT Token（备用键） | 同上 |
| `userInfo` | 用户信息（主键） | `{ id, name, role, ... }` |
| `user_info` | 用户信息（备用键） | 同上 |
| `openid` | 微信 OpenID | `oXXXXXXXXXXXXXXXXXXXXXXXXXXX` |

**为什么使用双键存储？**
- 兼容性：不同页面可能使用不同的 key
- 容错性：如果一个 key 被意外删除，还有备用 key

## ⚠️ 注意事项

### 1. 域名配置

**开发环境：**
- 在开发者工具中勾选"不校验合法域名"即可

**生产环境：**
- 必须在小程序后台配置 `https://crm.andejiazheng.com` 为合法域名
- 否则无法发起网络请求

### 2. Token 过期处理

Token 过期时会自动：
1. 清除本地存储的 Token 和用户信息
2. 显示提示："登录已过期，请重新登录"
3. 跳转到登录页面

### 3. 权限控制

用户信息中包含 `role` 和 `permissions` 字段，可用于权限控制：

```javascript
Page({
  onLoad() {
    const userInfo = wx.getStorageSync('userInfo');
    
    // 检查角色
    if (!['admin', 'manager'].includes(userInfo.role)) {
      wx.showToast({
        title: '无权限访问',
        icon: 'none'
      });
      wx.navigateBack();
    }
  }
});
```

## 🐛 常见问题

### Q1: 登录时提示"网络请求失败"

**原因：** 未关闭域名校验

**解决：**
1. 打开开发者工具的"详情" → "本地设置"
2. 勾选"不校验合法域名"

### Q2: 登录成功但没有跳转

**原因：** 可能是 Token 保存失败

**解决：**
1. 打开控制台查看日志
2. 检查是否有 `💾 保存认证数据` 的日志
3. 检查本地存储中是否有 `access_token`

### Q3: Token 验证失败

**原因：** Token 可能已过期或无效

**解决：**
1. 清除本地存储
2. 重新登录
