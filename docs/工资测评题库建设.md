
## 三、题目编写规范

> **重要**：技能题（skill）已升级为 **v2 学术格式**（题干→ABCD→渐变给分→explanation→source）。
> 硬件 / 心理（hardware / personality）保持 **v1 口语格式** 不变。两套规则并行，按 `section` 字段路由。

### 硬件 / 心理题规范（v1，不变）
- 题干 ≤ 25 字，选项 ≤ 12 字，全口语化
- **同题所有选项字数差 ≤ 3 字**——防止"最长选项 = 正确答案"
- **至少 30% 判断题**（实际可到 40-60%）
- 不要"以上都对/不对"等废选项
- 没有阿姨看不懂的术语（行业熟词如"恶露"可保留）
- 不写 `explanation` / `source` 字段

### 技能题规范（v2，2026-05 升级）
- 题干 ≤ 50 字，**全部 4 选项 choice**（skill 不再用 judge）
- 单选项 ≤ 25 字，**同题选项字数差 ≤ 8 字**
- 4 选项 score 必须满足：
  - 有且仅有 1 个 `score=10`（正确答案）
  - 其余 ∈ `{0, 3, 4, 7}`，渐变体现"错的程度"
  - 事实型题（顺序、剂量、阈值）多为 `[10,0,0,0]` 或 `[10,3,0,0]`
  - 操作型题（"宝宝呛奶处置"等）多铺满 `[10,7,3,0]`
- 必填新字段：
  - `explanation` ≤ 80 字——"为什么这个对、其他错在哪、临床警示是什么"
  - `source` ≤ 40 字——国标 / 教材出处（具体到版次或年份）
- 干扰项必须是**真实专业误区**（不能荒诞），照顾外行易踩坑
- 不允许"以上都对/不对"

### 评分梯度（0~10 分，云函数 `computeSectionScores` 已固化）
- 阿姨选了哪一项就拿那一项的 `score`
- 该题 `maxScore = max(options.score)`（v2 恒为 10；v1 判断题也基本为 10）
- 模块得分 = ∑题 score → 占 ∑maxScore 比例 → 乘模块权重 → 求和取整 = 总分
- v2 升级**不改算分逻辑**，只新增 `explanation` / `source` 字段透传到结果页

### JSONL Schema

**v1（hardware / personality）每行**：
```json
{
  "jobType": "yuexin",
  "section": "hardware",
  "type": "judge",
  "question": "您带过双胞胎吗？",
  "options": [
    {"text": "带过", "score": 10},
    {"text": "没带过", "score": 3}
  ]
}
```

**v2（skill）每行**：
```json
{
  "jobType": "yuexin",
  "section": "skill",
  "subsection": "产妇护理",
  "difficulty": "easy",
  "type": "choice",
  "question": "正常产妇产后恶露排出的顺序依次是？",
  "options": [
    {"text": "红色 → 白色 → 浆液性", "score": 0},
    {"text": "红色 → 浆液性 → 白色", "score": 10},
    {"text": "浆液性 → 红色 → 白色", "score": 3},
    {"text": "白色 → 浆液性 → 红色", "score": 0}
  ],
  "explanation": "恶露三阶段：①血性 1-4 天；②浆液性 5-10 天；③白色 10 天-3 周。红色超 2 周需就医。",
  "source": "《妇产科护理学》第 6 版（人卫版）"
}
```

字段说明：
- `section` ∈ `hardware | skill | personality`
- `difficulty` ∈ `easy | medium | hard`，**仅 skill 有**
- `type` = `judge`（2 选项，仅 v1）或 `choice`（4 选项）
- `explanation` / `source`：**仅 v2 必填**，v1 不写

## 四、算分与抽样算法

### 模块加权（已部署到云函数）
- 硬件 20 + 技能 60 + 心理 20 = 100 分
- 各模块完成率 × 权重 → 求和取整 = 总分
- 实现位置：`cloudfunctions/salaryAssessment/index.js → SECTION_WEIGHTS` + `computeSectionScores`

### 智能抽样（最后一批接入）
固定抽 30 题：硬件 4 + 技能 18 + 心理 8。

**技能 18 题按 `experienceYears` 落档**：
| 经验 | 易 | 中 | 难 |
|---|---|---|---|
| ≤ 2 年 | 11 | 5 | 2 |
| 3-5 年 | 5 | 9 | 4 |
| 6-10 年 | 3 | 8 | 7 |
| 10+ 年 | 2 | 6 | 10 |

每档内 **4 专项 round-robin** 抽，保证每个专项都被覆盖。心理 8 题在 5 子类间均衡随机；硬件 4 题随机。每道题选项落地前再洗牌一次重排 ABCD。
