# 标签匹配算法优化方案

## 🎯 最佳方案：AI 智能分类（推荐）

### 方案概述
使用 AI 大模型直接读取文章内容，进行智能分类。这是**最准确、最简单、最易维护**的方案。

### 核心优势 ✅
1. **准确度高** - AI 理解语义和上下文，不是简单的关键词匹配
2. **维护简单** - 无需维护复杂的正则表达式和关键词库
3. **自适应** - 可以处理新的表达方式和边缘情况
4. **多标签支持** - 可以准确判断文章属于多个分类
5. **成本可控** - 只在文章发布/更新时调用一次，不是每次展示都调用

### 实现方案

#### 方案 A：使用微信云开发 AI 能力（推荐）
你的项目已经集成了 DeepSeek-V3，可以直接使用！

```javascript
// cloudfunctions/articleService/index.js
const cloud = require('wx-server-sdk');
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV });

/**
 * 使用 AI 为文章智能分类
 * @param {Object} article - 文章对象 { title, summary, content }
 * @returns {Array<string>} - 标签数组，如 ['孕期呵护', '备孕好孕']
 */
async function classifyArticleByAI(article) {
  const { title, summary, content } = article;

  // 构建 prompt
  const prompt = `你是一个母婴育儿内容分类专家。请根据文章内容，从以下6个分类中选择最合适的1-2个标签：

【分类说明】
1. 备孕好孕 - 备孕准备、孕前检查、叶酸补充、排卵受孕等
2. 孕期呵护 - 孕期保健、产检、胎动、孕期饮食、孕期不适等
3. 产后恢复 - 月子护理、产后修复、哺乳催乳、产后抑郁等
4. 新生儿养护 - 0-3个月宝宝护理、新生儿喂养、黄疸、脐带护理等
5. 婴幼护理 - 4个月以上宝宝护理、辅食添加、生长发育、常见疾病等
6. 亲子早教 - 早教启蒙、亲子互动、绘本阅读、习惯培养等

【文章信息】
标题：${title}
摘要：${summary || '无'}
内容：${content ? content.substring(0, 500) : '无'}

【输出要求】
1. 只输出标签名称，用逗号分隔，如：孕期呵护,备孕好孕
2. 最多选择2个标签
3. 如果无法判断，输出：未分类
4. 不要输出任何解释说明`;

  try {
    // 调用 AI 接口
    const result = await cloud.openapi.ai.chat({
      model: 'deepseek-v3',
      messages: [{
        role: 'user',
        content: prompt
      }],
      temperature: 0.3,  // 降低随机性，提高一致性
      maxTokens: 50
    });

    const response = result.choices[0]?.message?.content || '';
    console.log('AI 分类结果:', response);

    // 解析返回的标签
    const tags = response
      .split(/[,，、]/)
      .map(t => t.trim())
      .filter(t => [
        '备孕好孕', '孕期呵护', '产后恢复',
        '新生儿养护', '婴幼护理', '亲子早教'
      ].includes(t));

    return tags.length > 0 ? tags : ['未分类'];

  } catch (error) {
    console.error('AI 分类失败:', error);
    // 降级到关键词匹配
    return fallbackClassify(article);
  }
}

/**
 * 降级方案：简单关键词匹配
 */
function fallbackClassify(article) {
  const text = `${article.title} ${article.summary}`.toLowerCase();

  const rules = [
    { keywords: ['备孕', '孕前', '叶酸', '排卵'], tag: '备孕好孕' },
    { keywords: ['孕期', '孕妇', '产检', '胎动'], tag: '孕期呵护' },
    { keywords: ['产后', '月子', '哺乳', '催乳'], tag: '产后恢复' },
    { keywords: ['新生儿', '满月', '黄疸', '脐带'], tag: '新生儿养护' },
    { keywords: ['辅食', '断奶', '湿疹', '发育'], tag: '婴幼护理' },
    { keywords: ['早教', '启蒙', '绘本', '亲子'], tag: '亲子早教' }
  ];

  for (const rule of rules) {
    if (rule.keywords.some(kw => text.includes(kw))) {
      return [rule.tag];
    }
  }

  return ['未分类'];
}

module.exports = { classifyArticleByAI };
```

#### 使用时机：在文章入库时自动分类

```javascript
// 在 CRM 后台文章发布时调用
async function publishArticle(articleData) {
  // 1. 使用 AI 自动分类
  const tags = await classifyArticleByAI({
    title: articleData.title,
    summary: articleData.summary,
    content: articleData.content
  });

  // 2. 保存到数据库
  articleData.tags = tags;
  articleData.primaryTag = tags[0];

  // 3. 保存文章
  await db.collection('articles').add({
    data: articleData
  });

  console.log(`文章《${articleData.title}》已分类为: ${tags.join(', ')}`);
}
```

## 📊 当前算法分析（关键词方案）

### 现有实现概述
当前的标签匹配算法位于 `miniprogram/pages/articleList/index.js` 中的 `extractCanonicalTags()` 函数。

### 核心流程
1. **结构化字段提取** - 从多个字段中提取标签（tags, category, keywords等）
2. **标签归一化** - 过滤噪声、统一同义词
3. **文本推断** - 基于标题和摘要的正则匹配
4. **评分机制** - 结构化标签+20分，标题匹配+6分，摘要匹配+3分
5. **标签选择** - 选择主标签+1个副标签（副标签需达到主标签50%分数）

### 当前问题 ⚠️

#### 1. 正则表达式性能问题
- 单个正则包含20+个关键词，匹配效率低
- 每篇文章需要对6个标签定义 × 2个字段（标题+摘要）= 12次正则匹配
- 列表页加载10篇文章 = 120次正则匹配

#### 2. 缺少上下文语义理解
- 纯关键词匹配，无法理解语义
- 例如："如何避免孕期不适" vs "孕期不适的原因" - 都匹配"孕期不适"，但重点不同
- 无法处理否定语境："不是新生儿问题"仍会匹配"新生儿"

#### 3. 维护成本高
- 需要持续维护关键词库
- 新的表达方式需要手动添加
- 不同标签的关键词数量不均衡
