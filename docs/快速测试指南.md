## 🚀 测试步骤

### 测试 1：账号密码登录

#### 步骤：

1. **启动小程序**
   - 在开发者工具中点击"编译"
   - 等待小程序加载完成

2. **进入登录页面**
   - 如果已登录，先退出登录
   - 点击底部 Tab "个人中心"
   - 点击"登录"按钮

3. **打开账号密码登录弹窗**
   - 点击"账号密码登录"按钮
   - 弹窗应该显示出来

4. **输入账号密码**
   - 账号：输入"安得家政"的账号
   - 密码：输入"安得家政"的密码

5. **点击登录**
   - 点击"登录"按钮
   - 观察控制台日志

#### 预期结果：

✅ **成功标志：**
- 控制台显示：`🔐 开始账号密码登录: xxx`
- 控制台显示：`📡 登录响应: { success: true, ... }`
- 控制台显示：`💾 保存认证数据: { hasToken: true, ... }`
- 控制台显示：`✅ 登录成功: xxx`
- 显示提示："登录成功"
- 自动跳转到首页

❌ **失败标志：**
- 显示提示："账号或密码错误"
- 显示提示："网络请求失败"
- 控制台显示错误日志

---

### 测试 2：Token 验证

#### 步骤：

1. **登录成功后**
   - 确保已经登录成功
   - 在首页停留

2. **重新进入登录页面**
   - 点击底部 Tab "个人中心"
   - 点击"登录"按钮（如果显示）

3. **观察行为**
   - 观察是否自动跳转到首页
   - 观察控制台日志

#### 预期结果：

✅ **成功标志：**
- 控制台显示：`🔍 检测到本地 Token，验证有效性...`
- 控制台显示：`✅ Token 有效，用户已登录，跳转到首页`
- 自动跳转到首页（不显示登录界面）

❌ **失败标志：**
- 显示登录界面
- 控制台显示：`❌ Token 验证失败`

---

### 测试 3：Token 过期处理

#### 步骤：

1. **手动清除 Token**
   - 打开控制台
   - 输入：`wx.removeStorageSync('access_token')`
   - 输入：`wx.removeStorageSync('token')`

2. **访问需要登录的页面**
   - 点击底部 Tab "简历列表"
   - 观察行为

#### 预期结果：

✅ **成功标志：**
- 显示提示："登录已过期，请重新登录"
- 自动跳转到登录页面

---

### 测试 4：本地存储检查

#### 步骤：

1. **登录成功后**
   - 确保已经登录成功

2. **打开控制台**
   - 点击"Console"标签

3. **检查本地存储**
   - 输入：`wx.getStorageSync('access_token')`
   - 输入：`wx.getStorageSync('userInfo')`

#### 预期结果：

✅ **成功标志：**
- `access_token` 返回一个长字符串（JWT Token）
- `userInfo` 返回用户信息对象：
  ```javascript
  {
    id: "user_123",
    name: "张三",
    username: "zhangsan",
    role: "admin",
    permissions: [...],
    ...
  }
  ```

## 🐛 常见问题排查

### 问题 1：登录时提示"网络请求失败"

**排查步骤：**
1. 检查是否勾选"不校验合法域名"
2. 检查网络连接是否正常
3. 检查"安得家政"的服务器是否正常

**解决方法：**
- 打开"详情" → "本地设置" → 勾选"不校验合法域名"

---

### 问题 2：登录时提示"账号或密码错误"

**排查步骤：**
1. 检查账号密码是否正确
2. 检查是否是"安得家政"的账号
3. 尝试在"安得家政"小程序中登录

**解决方法：**
- 使用正确的"安得家政"账号密码

---

### 问题 3：登录成功但没有跳转

**排查步骤：**
1. 查看控制台日志
2. 检查是否有错误信息
3. 检查本地存储中是否有 `access_token`

**解决方法：**
- 查看控制台日志，找到具体错误原因

---

### 问题 4：Token 验证失败

**排查步骤：**
1. 检查 Token 是否存在
2. 检查 Token 是否过期
3. 检查 API 是否正常

**解决方法：**
- 清除本地存储，重新登录
