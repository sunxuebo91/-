## 一、功能概述

### 1.1 产品定位
"宝贝日记"是安得褓贝小程序的核心服务记录功能，旨在**数字化替代传统纸质上户日记**，让月嫂阿姨每天记录宝宝和宝妈的各项健康指标、护理情况，实现：
- ✅ 数字化记录，永久保存，随时查看
- ✅ 标准化表单，规范服务流程
- ✅ 客户可实时查看，增强信任感
- ✅ 数据可追溯，服务质量可量化

### 1.2 服务周期
支持不同服务套餐的日记记录：
- **26天套餐**：新生儿护理基础套餐
- **42天套餐**：标准月子护理套餐
- **52天套餐**：延长护理套餐
- **78天套餐**：深度护理套餐

### 1.3 用户角色
| 角色 | 权限 | 使用场景 |
|------|------|----------|
| **月嫂阿姨（staff）** | 创建、编辑、查看日记 | 每天填写宝宝和宝妈的护理记录 |
| **客户（customer）** | 查看日记 | 实时了解宝宝和宝妈的护理情况 |

## 四、云函数设计

### 4.1 babyDiaryService 云函数

**路径**：`cloudfunctions/babyDiaryService/index.js`

**支持的 action**：

```javascript
{
  // 日记管理
  "createDiary",      // 创建日记
  "updateDiary",      // 更新日记
  "getDiary",         // 获取单条日记
  "listDiaries",      // 获取日记列表
  "deleteDiary",      // 删除日记

  // 合同管理
  "createContract",   // 创建服务合同
  "getContract",      // 获取合同详情
  "listContracts",    // 获取合同列表
  "updateContract",   // 更新合同

  // 统计分析
  "getDiaryStats",    // 获取日记统计
}
```

**权限控制**：
- 月嫂只能操作自己的日记
- 客户只能查看自己的日记
- 使用 isStaff 判断角色
