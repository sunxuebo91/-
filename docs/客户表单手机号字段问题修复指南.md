# 客户表单手机号字段问题修复指南

## 问题描述

根据用户反馈和截图分析：

1. **当前问题**：在"编辑客户"表单中，手机号字段显示为必填项（带红色星号 *）
2. **期望行为**：手机号应该是可选字段，不是必填项
3. **触发场景**：修改微信号后点击"保存"，系统弹出提示"请填写所有必填字段"，强制要求填写手机号
4. **核心问题**：前端表单验证逻辑错误地将 `phone` 字段标记为必填

## 问题分析

### API 接口现状

根据 `API完整文档.md` 和架构文档 `docs/安得家政架构分析-真实方案.md`，客户管理 API 端点为：

```
POST   /api/customers/miniprogram/create        # 创建客户
PATCH  /api/customers/miniprogram/:id           # 更新客户
```

### 需要检查的位置

由于客户编辑表单不在当前 miniprogram 代码库中，需要在以下位置进行检查和修复：

#### 1. 后端 API Schema 定义

**位置**：CRM 后端服务（`https://crm.andejiazheng.com`）

**需要检查**：
- Customer Schema/Model 定义中 `phone` 字段的 `required` 属性
- 验证规则（Validation Rules）中 `phone` 字段的必填验证

**修复示例**（Node.js/Express + Mongoose）：

```javascript
// ❌ 错误：phone 设置为必填
const customerSchema = new mongoose.Schema({
  name: { type: String, required: true },
  phone: { type: String, required: true },  // 需要改为 false
  wechat: { type: String },
  // ... 其他字段
});

// ✅ 正确：phone 设置为可选
const customerSchema = new mongoose.Schema({
  name: { type: String, required: true },
  phone: { type: String, required: false },  // 或者直接不设置 required
  wechat: { type: String },
  // ... 其他字段
});
```

#### 2. 前端表单验证

**位置**：CRM 管理后台前端（可能是 admin-web 或独立的管理系统）

**需要检查**：
- 表单字段定义中 `phone` 的 `required` 属性
- 表单提交前的客户端验证逻辑

**修复示例**（Vue/React）：

```javascript
// ❌ 错误：phone 标记为必填
const formRules = {
  name: [{ required: true, message: '请输入客户姓名' }],
  phone: [{ required: true, message: '请输入手机号' }],  // 需要移除或改为 false
  wechat: [{ required: false }],
};

// ✅ 正确：phone 设置为可选
const formRules = {
  name: [{ required: true, message: '请输入客户姓名' }],
  phone: [{ required: false }],  // 或者完全移除这个规则
  wechat: [{ required: false }],
};
```

**HTML 模板修复**：

```html
<!-- ❌ 错误：显示必填星号 -->
<label>手机号 *</label>
<input v-model="form.phone" required />

<!-- ✅ 正确：移除必填标记 -->
<label>手机号</label>
<input v-model="form.phone" />
```

#### 3. API 端点验证逻辑

**位置**：CRM 后端 API 路由处理器

**需要检查**：
- `POST /api/customers/miniprogram/create` 的请求体验证
- `PATCH /api/customers/miniprogram/:id` 的请求体验证

**修复示例**（Express + express-validator）：

```javascript
// ❌ 错误：phone 设置为必填
router.post('/customers/miniprogram/create', [
  body('name').notEmpty().withMessage('客户姓名不能为空'),
  body('phone').notEmpty().withMessage('手机号不能为空'),  // 需要移除或改为可选
], createCustomer);

// ✅ 正确：phone 设置为可选
router.post('/customers/miniprogram/create', [
  body('name').notEmpty().withMessage('客户姓名不能为空'),
  body('phone').optional().isMobilePhone('zh-CN'),  // 可选，但如果提供则验证格式
], createCustomer);
```

## 修复步骤

### 步骤 1：确认 API Schema

```bash
# 测试创建客户时不提供 phone 字段
curl -X POST https://crm.andejiazheng.com/api/customers/miniprogram/create \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "测试客户",
    "wechat": "test_wechat"
  }'
```

**预期结果**：
- ✅ 成功创建客户（返回 200/201）
- ❌ 如果返回 400 错误提示 "phone is required"，则需要修复后端 Schema

### 步骤 2：修复后端 Schema

1. 找到 Customer Model 定义文件
2. 将 `phone` 字段的 `required` 属性改为 `false` 或移除
3. 更新数据库迁移（如果需要）
4. 重启后端服务

### 步骤 3：修复前端表单

1. 找到客户编辑表单组件
2. 移除 `phone` 字段的必填验证规则
3. 移除 HTML 中的必填星号 `*`
4. 移除 `required` 属性
5. 重新构建并部署前端

### 步骤 4：测试验证

测试以下场景：

1. ✅ 创建客户时不填写手机号 → 应该成功
2. ✅ 创建客户时只填写微信号 → 应该成功
3. ✅ 编辑客户时清空手机号 → 应该成功
4. ✅ 编辑客户时修改微信号但不填手机号 → 应该成功
