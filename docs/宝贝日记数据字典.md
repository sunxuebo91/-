## 一、babyDiaries 集合

### 1.1 集合说明
- **集合名称**：babyDiaries
- **用途**：存储月嫂每日填写的宝宝和宝妈护理记录
- **权限**：仅创建者和相关客户可读，仅创建者可写

### 1.2 字段定义

| 字段名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| _id | String | 是 | 文档唯一标识 | "diary_20260127_001" |
| contractId | String | 是 | 关联的服务合同ID | "contract_001" |
| customerId | String | 是 | 客户ID（宝妈的 openid） | "oXXXX" |
| nurseId | String | 是 | 月嫂ID（阿姨的 openid） | "oYYYY" |
| serviceDate | String | 是 | 服务日期（YYYY-MM-DD） | "2026-01-27" |
| dayNumber | Number | 是 | 服务第几天（1-78） | 15 |
| baby | Object | 是 | 宝宝记录对象 | 见下表 |
| mother | Object | 是 | 宝妈记录对象 | 见下表 |
| status | String | 是 | 状态（draft/published） | "published" |
| createdAt | Date | 是 | 创建时间 | Date() |
| updatedAt | Date | 是 | 更新时间 | Date() |
| createdBy | String | 是 | 创建者 openid | "oYYYY" |

### 1.3 baby 对象结构

#### 1.3.1 基础指标（baby.basics）

| 字段名 | 类型 | 必填 | 说明 | 单位 | 正常范围 |

| weight | Number | 否 | 体重 | kg | 2.5-5.0 |
| height | Number | 否 | 身高 | cm | 45-55 |

| formulaFeeding.times | Number | 否 | 奶粉次数 | 2 |

#### 1.3.6 其他记录

| 字段名 | 类型 | 必填 | 说明 | 示例值 |

## 三、索引设计

### 3.1 babyDiaries 集合索引

| 索引名称 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| idx_contract | contractId | 单字段 | 按合同查询日记 |
| idx_nurse_date | nurseId + serviceDate | 复合 | 月嫂查询某日日记 |
| idx_customer_date | customerId + serviceDate | 复合 | 客户查询某日日记 |
| idx_date | serviceDate | 单字段 | 按日期查询 |

### 3.2 serviceContracts 集合索引

| 索引名称 | 字段 | 类型 | 说明 |
|----------|------|------|------|
| idx_nurse_status | nurseId + status | 复合 | 查询月嫂的活跃合同 |
| idx_customer_status | customerId + status | 复合 | 查询客户的活跃合同 |
| idx_status | status | 单字段 | 按状态查询 |

## 四、数据校验规则

### 4.1 必填字段校验
- 所有标记为"必填"的字段必须提供
- 字符串类型不能为空字符串
- 数字类型必须为有效数字

### 4.2 数据类型校验
- 日期格式：YYYY-MM-DD
- 体温范围：35.0-42.0℃
- 血压格式：数字/数字（如 120/80）
- 脉搏范围：40-150 次/分

### 4.3 业务规则校验
- serviceDate 必须在合同的 startDate 和 endDate 之间
- dayNumber 必须在 1 到 serviceDays 之间
- 同一合同的同一天只能有一条日记记录
- 只有月嫂本人可以创建和编辑日记
