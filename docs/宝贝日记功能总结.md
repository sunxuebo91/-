## 🎯 核心功能设计

### 1. 功能定位
**数字化替代传统纸质上户日记**，让月嫂阿姨每天记录宝宝和宝妈的各项健康指标、护理情况。

### 2. 核心价值
- ✅ **数字化记录**：永久保存，随时查看
- ✅ **标准化表单**：规范服务流程
- ✅ **实时共享**：客户可实时查看，增强信任感
- ✅ **数据可追溯**：服务质量可量化

### 3. 服务周期
支持 **26天、42天、52天、78天** 四种服务套餐。

### 4. 用户角色
- **月嫂阿姨（staff）**：创建、编辑、查看日记
- **客户（customer）**：查看日记

---

## 🗄️ 数据模型设计

### 1. 核心集合

#### babyDiaries（日记集合）
存储每日的宝宝和宝妈护理记录。

**核心字段**：
- 关联信息：contractId, customerId, nurseId
- 日期信息：serviceDate, dayNumber
- 记录内容：baby（宝宝记录）, mother（宝妈记录）
- 状态管理：status（draft/published）

#### serviceContracts（合同集合）
管理服务合同和周期。

**核心字段**：
- 关联信息：customerId, nurseId
- 服务信息：serviceType, serviceDays, startDate, endDate
- 宝宝信息：babyInfo
- 状态管理：status（active/completed/cancelled）

### 2. 数据关系
```
users (用户)
  ├── serviceContracts (服务合同)
  │     └── babyDiaries (日记记录)
  └── babyDiaries (创建的日记)
```

### 3. 索引设计
- babyDiaries：contractId, nurseId+serviceDate, customerId+serviceDate
- serviceContracts：nurseId+status, customerId+status

## 🎨 页面设计

### 1. 首页入口
**位置**：首页 Banner 下方，四个服务入口之后

**设计特点**：
- 渐变背景：粉紫色渐变（#FFE8F0 → #EDE7FF）
- 可爱的宝宝日记本图标
- 动态提示：显示今日已记录条数
- 仅 staff 角色可见

### 2. 日记列表页
**功能**：
- 显示当前服务中的所有家庭
- 显示服务进度（第X/Y天）
- 显示今日记录状态
- 快速进入填写或查看

### 3. 日记填写页
**设计原则**：
- 分段式表单（宝宝记录 + 宝妈记录）
- 常用项快速录入，非必填项可跳过
- 支持保存草稿，随时继续编辑
- 照片上传，记录宝宝成长瞬间

**表单内容**：

#### 宝宝记录
- 基础指标：体重、身高、头围、体温
- 喂养记录：母乳、奶粉、喝水
- 排泄记录：小便、大便
- 睡眠记录：时长、质量
- 护理记录：洗澡、脐带、皮肤、黄疸
- 照片上传
- 备注

#### 宝妈记录
- 基础指标：体温、血压、脉搏
- 恶露情况：量、颜色、气味
- 乳房护理：状况、奶量、护理措施
- 饮食记录：三餐 + 加餐 + 饮水
- 排泄记录：排尿、排便
- 情绪与睡眠
- 伤口恢复（剖腹产）
- 备注

### 4. 日记详情页
**功能**：
- 展示日记详情
- 支持编辑（阿姨端）
- 支持删除（阿姨端）
- 支持照片预览

## ☁️ 云函数设计

### babyDiaryService 云函数

**支持的操作**：

| Action | 说明 | 权限 |
|--------|------|------|
| createDiary | 创建日记 | 月嫂 |
| updateDiary | 更新日记 | 月嫂 |
| getDiary | 获取日记详情 | 月嫂/客户 |
| listDiaries | 获取日记列表 | 月嫂/客户 |
| deleteDiary | 删除日记 | 月嫂 |
| createContract | 创建服务合同 | 管理员 |
| listContracts | 获取合同列表 | 月嫂/客户 |
| updateContract | 更新合同 | 管理员 |
| getDiaryStats | 获取日记统计 | 月嫂/客户 |

**权限控制**：
- 使用 isStaff 判断角色
- 月嫂只能操作自己的日记
- 客户只能查看自己的日记

## 🎓 设计思路总结

作为一名资深的家政产品设计师，我在设计"宝贝日记"功能时，遵循了以下原则：

### 1. 以用户为中心
- 深入理解月嫂阿姨的工作场景和痛点
- 简化填写流程，降低学习成本
- 提供快捷操作，提高工作效率

### 2. 数据驱动
- 设计完整的数据模型，确保数据可追溯
- 预留扩展字段，支持未来功能迭代
- 建立索引，优化查询性能

### 3. 安全可靠
- 严格的权限控制，保护用户隐私
- 数据校验，确保数据质量
- 自动保存，防止数据丢失

### 4. 可扩展性
- 模块化设计，便于功能扩展
- 预留统计分析接口，支持未来数据分析
- 支持多种服务周期，适应不同业务场景
