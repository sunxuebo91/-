# 文章接口调试指南

## 🎯 目的

帮助你快速诊断文章接口是否正常工作，并找出问题所在。

## 📱 使用测试页面

### 步骤 1: 打开测试页面

1. 在微信开发者工具中打开小程序项目
2. 测试页面会自动作为首页打开（已设置为第一个页面）
3. 或者在任意页面的控制台输入：
   ```javascript
   wx.navigateTo({ url: '/pages/test-article-api' })
   ```

### 步骤 2: 运行测试

点击 **"🚀 测试所有接口"** 按钮，会依次测试：

1. ✅ **Banner 接口**（对比测试，这个接口是正常的）
   - `GET https://crm.andejiazheng.com/api/banners/miniprogram/active`
   
2. 📰 **文章列表接口**（需要测试的）
   - `GET https://crm.andejiazheng.com/api/articles?status=published&page=1&pageSize=10`
   
3. 📄 **文章详情接口**（需要测试的）
   - `GET https://crm.andejiazheng.com/api/articles/:id`

### 步骤 3: 查看结果

测试结果会显示在页面上，包括：
- ✅ 接口响应状态码
- 📦 完整的响应数据
- ⚠️ 错误信息（如果有）

### 步骤 4: 复制日志

点击 **"📋 复制日志"** 按钮，将测试结果复制到剪贴板，然后发给我。

## 🔍 可能的测试结果

### 结果 A: 接口不存在（404）

```
❌ 接口请求失败: request:fail
或
✅ 接口响应成功 (状态码: 404)
```

**原因**: 后端服务器上没有部署文章接口

**解决方案**: 
1. 在后端服务器上添加文章接口
2. 或者创建文章云函数（推荐）

### 结果 B: 接口存在但返回空数据

```
✅ 接口响应成功 (状态码: 200)
📦 响应数据: { "success": true, "data": { "list": [], "total": 0 } }
📰 获取到 0 篇文章
```

**原因**: 数据库中没有文章数据

**解决方案**: 
1. 在 CRM 后台添加文章
2. 确保文章状态为"已发布"

### 结果 C: 接口正常工作

```
✅ 接口响应成功 (状态码: 200)
📦 响应数据: { "success": true, "data": { "list": [...], "total": 10 } }
📰 获取到 10 篇文章
```

**原因**: 一切正常！

**下一步**: 检查小程序页面为什么没有显示文章

### 结果 D: 网络错误

```
❌ 接口请求失败: request:fail url not in domain list
```

**原因**: 域名未配置到小程序合法域名列表

**解决方案**: 
1. 在微信公众平台配置服务器域名
2. 或在开发者工具中勾选"不校验合法域名"

## 🛠️ 手动测试（备选方案）

如果测试页面无法使用，可以在控制台手动运行：

```javascript
// 测试文章列表接口
wx.request({
  url: 'https://crm.andejiazheng.com/api/articles?status=published&page=1&pageSize=10',
  method: 'GET',
  header: {
    'Content-Type': 'application/json',
    'X-Client-Type': 'miniprogram',
    'X-Platform': 'wechat'
  },
  success: (res) => {
    console.log('✅ 成功:', res);
  },
  fail: (err) => {
    console.error('❌ 失败:', err);
  }
});
```
