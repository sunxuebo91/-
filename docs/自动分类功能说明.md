# 自动分类功能说明

## 🎯 功能概述

**小程序在接收 CRM 文章时自动添加 AI 分类标签**

无需修改 CRM 后台，文章从 CRM 获取后，小程序会自动调用 AI 进行智能分类并添加标签。

---

## ✨ 工作原理

```
CRM 后台发布文章（无标签）
         ↓
小程序获取文章列表/详情
         ↓
检测文章是否有有效标签
         ↓
    没有标签？
         ↓
自动调用 AI 分类云函数
         ↓
为文章添加 AI 标签
         ↓
显示带标签的文章
```

---

## 📝 实现细节

### 1. 自动分类触发时机

**文章列表页面**：
- 调用 `getArticleList()` 时自动为所有文章分类
- 并发处理，速度快

**文章详情页面**：
- 调用 `getArticleDetail()` 时自动分类
- 确保详情页也有正确标签

### 2. 智能跳过机制

**已有有效标签的文章会被跳过**，避免重复分类：

有效标签包括：
- ✅ 备孕好孕
- ✅ 孕期呵护
- ✅ 产后恢复
- ✅ 新生儿养护
- ✅ 婴幼护理
- ✅ 亲子早教

### 3. 并发处理优化

使用 `Promise.all()` 并发处理多篇文章，提高速度：

```javascript
// 同时为 10 篇文章分类，而不是一篇一篇处理
const classifyPromises = articles.map(article => autoClassifyArticle(article));
await Promise.all(classifyPromises);
```

## 🚀 使用方法

### 默认行为（自动分类）

```javascript
// 获取文章列表 - 自动分类
const result = await articleService.getArticleList({
  page: 1,
  pageSize: 10
});

// 获取文章详情 - 自动分类
const detail = await articleService.getArticleDetail(articleId);
```

### 禁用自动分类（可选）

如果某些场景不需要自动分类：

```javascript
// 获取文章列表 - 不自动分类
const result = await articleService.getArticleList({
  page: 1,
  pageSize: 10,
  autoClassify: false  // ← 禁用自动分类
});

// 获取文章详情 - 不自动分类
const detail = await articleService.getArticleDetail(articleId, false);
```

### 手动分类单篇文章

```javascript
const { autoClassifyArticle } = require('../../services/article.js');

// 手动为文章分类
const tags = await autoClassifyArticle({
  title: '孕期如何补充叶酸',
  summary: '叶酸是孕前和孕早期必须补充的营养素',
  content: '备孕期间，建议每天补充400微克叶酸...'
});

console.log('分类结果:', tags);  // ['备孕好孕', '孕期呵护']
```

## 💰 成本分析

### 单次分类成本

- **单篇文章**: 约 0.0001 元（1分钱可分类100篇）
- **10篇文章**: 约 0.001 元
- **100篇文章**: 约 0.01 元

### 实际使用场景

**场景 1：用户浏览文章列表**
- 每次加载 10 篇文章
- 假设 5 篇已有标签（跳过），5 篇需要分类
- 成本：0.0005 元/次

**场景 2：用户查看文章详情**
- 单篇文章分类
- 如果已有标签则跳过
- 成本：0 元（跳过）或 0.0001 元（分类）

**月度估算**：
- 假设每天 100 个用户，每人浏览 5 次列表
- 每次 10 篇文章，平均 5 篇需要分类
- 每天：100 × 5 × 5 × 0.0001 = 0.25 元
- **每月：约 7.5 元**

### 成本优化建议

1. **缓存机制**（推荐）
   - 将分类结果保存到本地缓存
   - 相同文章不重复分类
   - 可降低成本 80%+

2. **批量预分类**（可选）
   - 定期批量为所有文章分类
   - 将标签保存到 CRM 数据库
   - 小程序直接使用已有标签

## 🔧 配置选项

### 修改自动分类行为

编辑 `miniprogram/services/article.js`：

```javascript
// 修改有效标签列表
const validTags = ['备孕好孕', '孕期呵护', '产后恢复', '新生儿养护', '婴幼护理', '亲子早教'];

// 修改跳过逻辑
if (article.tags && Array.isArray(article.tags) && article.tags.length > 0) {
  const hasValidTag = article.tags.some(tag => validTags.includes(tag));
  if (hasValidTag) {
    return article.tags;  // 跳过分类
  }
}
```

## ⚠️ 注意事项

### 1. 首次加载可能较慢

第一次加载文章列表时，需要为所有文章分类，可能需要 2-5 秒。

**解决方案**：
- 显示加载提示
- 使用骨架屏
- 后续加载会跳过已分类的文章

### 2. 网络问题处理

如果 AI 分类失败（网络问题、AI 服务不可用），会自动降级到关键词匹配。

### 3. 成本控制

如果担心成本，可以：
- 禁用自动分类：`autoClassify: false`
- 只在文章详情页分类
- 使用本地缓存
