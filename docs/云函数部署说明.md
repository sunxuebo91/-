# 云函数部署说明

> **重要**: 修改云函数代码后，必须重新部署才能生效！

---

## 🚀 部署步骤

### 1. 打开微信开发者工具

### 2. 找到云函数目录

在左侧文件树中找到 `cloudfunctions/userService` 文件夹

### 3. 右键点击 `userService` 文件夹

### 4. 选择"上传并部署：云端安装依赖"

**注意**: 
- ✅ 选择"云端安装依赖"（推荐）
- ❌ 不要选择"上传并部署：不安装依赖"

### 5. 等待部署完成

部署过程大约需要 30 秒到 1 分钟，控制台会显示：

```
[云函数] [userService] 上传成功
[云函数] [userService] 云端安装依赖中...
[云函数] [userService] 云端安装依赖完成
[云函数] [userService] 部署成功
```

### 6. 验证部署

部署完成后，可以在控制台看到：

```
✅ [云函数] [userService] 部署成功
```

---

## 🎯 修改目的

**问题**: 云函数返回的用户对象中没有 `_openid` 字段，导致小程序端无法获取 OpenID，进而无法调用 CRM 后端接口。

**解决方案**: 在云函数返回用户对象时，显式添加 `_openid` 字段，确保小程序端能够获取到 OpenID。

## ⚠️ 常见问题

### Q1: 部署失败，提示"云函数不存在"

**解决方案**: 
1. 确保已经在云开发控制台创建了 `userService` 云函数
2. 或者先右键点击 `userService` 文件夹，选择"创建并部署：云端安装依赖"

### Q2: 部署成功，但代码没有生效

**解决方案**: 
1. 等待 1-2 分钟，云函数部署需要时间
2. 重新编译小程序
3. 清除缓存后重新打开小程序

### Q3: 控制台仍然显示"获取 OpenID 失败"

**解决方案**: 
1. 确认云函数已经部署成功
2. 检查云函数日志，查看是否有错误
3. 重新编译小程序
