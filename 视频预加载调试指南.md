# 🔍 视频预加载调试指南

## 📋 检查清单

### 1️⃣ 检查视频 URL 是否正确提取

打开列表页，查看控制台：
```
✅ 应该看到：
🎬 简历有视频: 67890abcdef... 张三 https://...

❌ 如果没有，说明：
- API 返回的数据中没有 selfIntroductionVideo 字段
- 或者 selfIntroductionVideo.url 为空
```

**解决方案**：检查 API 返回的数据结构


### 2️⃣ 检查预加载是否启动

列表加载完成后，应该看到：
```
✅ 应该看到：
🚀 列表加载完成，开始预加载视频...
📥 开始预加载视频，共 X 个
📥 预加载第 1 批: 3 个

❌ 如果没有，说明：
- 列表中没有视频
- 或者 preloadAllVideos() 没有被调用
```


### 3️⃣ 检查 cloud:// URL 转换

如果视频 URL 是 `cloud://` 格式：
```
✅ 应该看到：
🔄 转换 cloud:// URL: 67890abcdef...
✅ cloud:// URL 转换成功: 67890abcdef... https://...

❌ 如果看到：
❌ cloud:// URL 转换失败
说明：wx.cloud.getTempFileURL 调用失败
```

**解决方案**：
- 检查是否初始化了云开发
- 检查 app.js 中是否有 `wx.cloud.init()`


### 4️⃣ 检查视频下载

每个视频下载时应该看到：
```
✅ 应该看到：
📥 开始预加载视频: 67890abcdef...
   URL: https://...
📊 下载响应: { statusCode: 200, duration: 2000ms, tempFilePath: ... }
✅ 视频预加载成功: 67890abcdef... 耗时: 2000ms

❌ 如果看到：
❌ 下载失败，状态码: 403
说明：视频域名未加入下载白名单
```

**解决方案**：
1. 打开小程序后台
2. 进入 "开发" -> "开发管理" -> "开发设置"
3. 找到 "服务器域名" -> "downloadFile 合法域名"
4. 添加视频域名（例如：`https://xxx.tcb.qcloud.la`）


### 5️⃣ 检查详情页是否使用预加载视频

点击列表卡片进入详情页：
```
✅ 应该看到：
✅ 使用列表页预加载的视频: wxfile://...
✅ 使用预加载视频路径: wxfile://...

❌ 如果没有，说明：
- 列表页预加载失败
- 或者详情页没有正确获取列表页数据
```


## 🐛 常见问题

### 问题 1：视频下载很慢
**原因**：视频文件太大（>10MB）

**解决方案**：
- 压缩视频（建议 <5MB）
- 降低分辨率（720p 足够）
- 降低码率（1-2Mbps）


### 问题 2：下载失败 403
**原因**：视频域名未加入白名单

**解决方案**：
```javascript
// 在小程序后台添加 downloadFile 合法域名
// 例如：https://xxx.tcb.qcloud.la
```


### 问题 3：cloud:// URL 转换失败
**原因**：云开发未初始化

**解决方案**：
```javascript
// 在 app.js 中确保有：
wx.cloud.init({
  env: 'your-env-id',
  traceUser: true
});
```


### 问题 4：预加载成功但详情页还是慢
**原因**：详情页没有使用预加载的本地路径

**检查**：
```javascript
// 在 resumeDetail/index.js 的 onLoad 中
console.log('预加载路径:', this.preloadedVideoPath);

// 在 setData 时
console.log('视频源:', detailWithAvatar.videoLocalSrc || detailWithAvatar.videoFileId);
```


## 🔧 调试技巧

### 1. 查看缓存状态
在控制台输入：
```javascript
// 查看当前缓存的视频数量
console.log('缓存数量:', videoPreloader.cache.size);

// 查看正在下载的视频
console.log('下载中:', videoPreloader.downloading.size);
```


### 2. 手动触发预加载
在控制台输入：
```javascript
// 获取当前页面实例
const page = getCurrentPages()[getCurrentPages().length - 1];

// 手动触发预加载
page.preloadAllVideos();
```


### 3. 清空缓存重新测试
```javascript
videoPreloader.clear();
```


## 📊 性能监控

### 关键指标
- **下载耗时**：每个视频应该在 2-5 秒内完成
- **缓存命中率**：详情页应该 100% 使用预加载视频
- **内存占用**：缓存 15 个视频约占用 50-100MB


### 优化建议
1. **视频大小**：控制在 3-5MB
2. **并发数**：根据网络情况调整（2-3 个）
3. **缓存数量**：根据内存情况调整（10-20 个）


## ✅ 验收标准

1. ✅ 列表加载完成后，自动开始预加载
2. ✅ 控制台能看到完整的预加载日志
3. ✅ 点击卡片进入详情页，视频秒开（<0.5秒）
4. ✅ 滚动列表，新加载的页面也会自动预加载
5. ✅ 缓存管理正常，不会无限增长


## 🎯 下一步

如果预加载还是很慢，可能需要：
1. 检查视频文件大小和格式
2. 检查网络环境（Wi-Fi vs 4G）
3. 考虑使用 CDN 加速
4. 考虑服务端转码优化
