# 🚀 视频预加载优化方案

## 📊 问题分析

### 原有问题
1. ❌ 视频只在详情页加载 - 用户点击后才开始下载
2. ❌ 没有预加载机制 - 每次都要等待下载
3. ❌ 用户体验差 - 播放卡顿，等待时间长

## ✅ 优化方案：智能视口预加载

### 核心思路
**在列表页自动预加载用户可见区域的视频，滚动时动态加载新视频**

### 技术实现

#### 1. **VideoPreloader 类** - 视频预加载管理器
```javascript
class VideoPreloader {
  - cache: Map          // 缓存已下载的视频
  - downloading: Set    // 正在下载的视频
  - maxCache: 15        // 最多缓存 15 个视频
  
  方法：
  - preload()           // 预加载单个视频
  - batchPreload()      // 批量预加载（限制并发）
  - getCached()         // 获取缓存的视频
  - _cleanOldCache()    // 清理旧缓存（FIFO）
}
```

#### 2. **IntersectionObserver** - 视口监听
- 监听 `.resume-card` 元素进入视口
- 当卡片 10% 进入可见区域时触发预加载
- 自动处理滚动场景

#### 3. **首屏预加载**
- 列表加载完成后，自动预加载前 5 个视频
- 限制并发数为 2，避免网络拥堵

#### 4. **详情页优先使用预加载视频**
- 从列表页获取已预加载的视频路径
- 直接使用本地路径，无需等待下载

## 🎯 优化效果

### 用户体验提升
- ✅ **秒开视频** - 从列表进入详情页，视频立即播放
- ✅ **流畅滚动** - 滚动列表时自动预加载，无感知
- ✅ **智能缓存** - 最多缓存 15 个视频，自动清理旧缓存
- ✅ **节省流量** - 只预加载可见区域的视频

### 技术优势
- ✅ **并发控制** - 最多 2 个并发下载，不阻塞主线程
- ✅ **内存管理** - FIFO 策略自动清理旧缓存
- ✅ **容错处理** - 下载失败自动回退到远程播放
- ✅ **跨页面共享** - 列表页和详情页共享缓存

// 1. 设置视口观察器
setupIntersectionObserver() {
  this.intersectionObserver = wx.createIntersectionObserver(this, {
    thresholds: [0.1]
  });
  
  this.intersectionObserver.relativeTo('.page').observe('.resume-card', (res) => {
    if (res.intersectionRatio > 0) {
      const { resumeId, videoUrl } = res.dataset;
      this.preloadVideoForResume(resumeId, videoUrl);
    }
  });
}
