# 员工评价功能调试说明

## 问题描述
评价统计卡片（红圈部分）不显示，但评价列表正常显示。

## 可能的原因

### 1. 统计接口返回数据为空或格式不对
- 检查统计接口是否正常返回数据
- 检查返回的字段名是否与代码中使用的一致

### 2. 显示条件不满足
- 统计卡片的显示条件：`wx:if="{{evaluations.statistics && evaluations.statistics.totalCount > 0}}"`
- 如果 `totalCount` 为 0 或 `statistics` 为 `null`，卡片不会显示

## 调试步骤

### 步骤 1：查看控制台日志
在小程序开发者工具中打开控制台，查找以下日志：

```
⭐ 开始加载员工评价, employeeId: xxx
⭐ 评价统计响应: {...}
⭐ 评价列表响应: {...}
⭐ 统计数据原始值: {...}
⭐ 统计数据处理后: {...}
⭐ 评价数据加载完成: {...}
```

### 步骤 2：检查统计接口返回数据
重点关注 `⭐ 评价统计响应` 这条日志，查看：
- `success` 是否为 `true`
- `data` 是否存在
- `data` 中包含哪些字段

### 步骤 3：检查字段名映射
代码已经兼容多种字段名格式：

| 显示名称 | 可能的字段名 |
|---------|-------------|
| 总评价数 | `totalCount`, `total` |
| 综合评分 | `averageRating`, `averageOverallRating`, `average_rating` |
| 服务态度 | `averageServiceAttitude`, `averageServiceAttitudeRating`, `average_service_attitude` |
| 专业技能 | `averageProfessionalSkill`, `averageProfessionalSkillRating`, `average_professional_skill` |
| 工作效率 | `averageWorkEfficiency`, `averageWorkEfficiencyRating`, `average_work_efficiency` |
| 沟通能力 | `averageCommunication`, `averageCommunicationRating`, `average_communication` |

如果实际字段名不在上述列表中，需要修改代码添加新的字段名映射。

### 步骤 4：检查 API 接口
确认以下接口是否正常工作：

**统计接口**：
```
GET https://crm.andejiazheng.com/api/employee-evaluations/miniprogram/statistics/{employeeId}
```

**列表接口**：
```
GET https://crm.andejiazheng.com/api/employee-evaluations/miniprogram/list?employeeId={id}&page=1&pageSize=5
```

### 步骤 5：手动测试接口
使用 Postman 或浏览器直接访问统计接口，查看返回数据格式：

```bash
# 替换 {employeeId} 为实际的员工ID
curl https://crm.andejiazheng.com/api/employee-evaluations/miniprogram/statistics/{employeeId}
```

## 常见问题解决方案

### 问题 1：统计接口返回 404
**原因**：接口路径不正确或后端未实现该接口

**解决方案**：
1. 检查后端是否已实现统计接口
2. 确认接口路径是否正确
3. 如果接口不存在，需要后端先实现该接口

### 问题 2：统计接口返回空数据
**原因**：该员工没有评价数据

**解决方案**：
1. 先创建一些测试评价数据
2. 或者修改显示逻辑，即使没有统计数据也显示默认的 0 分

### 问题 3：字段名不匹配
**原因**：后端返回的字段名与前端代码中使用的不一致

**解决方案**：
修改 `miniprogram/pages/resumeDetail/index.js` 中的字段映射代码，添加实际的字段名。

例如，如果后端返回的字段名是 `avg_rating`，需要添加：
```javascript
const averageRating = data.averageRating || data.averageOverallRating || data.average_rating || data.avg_rating || 0;
```

## 临时解决方案

如果统计接口暂时无法使用，可以临时从评价列表中计算统计数据：

修改 `miniprogram/pages/resumeDetail/index.js` 中的 `loadEvaluations` 方法，在评价列表加载成功后，手动计算统计数据：

```javascript
// 如果统计接口失败，从列表中计算统计数据
if (!statistics && list.length > 0) {
  const totalRating = list.reduce((sum, item) => sum + (item.overallRating || 0), 0);
  const totalServiceAttitude = list.reduce((sum, item) => sum + (item.serviceAttitudeRating || 0), 0);
  const totalProfessionalSkill = list.reduce((sum, item) => sum + (item.professionalSkillRating || 0), 0);
  const totalWorkEfficiency = list.reduce((sum, item) => sum + (item.workEfficiencyRating || 0), 0);
  const totalCommunication = list.reduce((sum, item) => sum + (item.communicationRating || 0), 0);
  
  statistics = {
    totalCount: list.length,
    averageRating: (totalRating / list.length).toFixed(1),
    averageServiceAttitude: (totalServiceAttitude / list.length).toFixed(1),
    averageProfessionalSkill: (totalProfessionalSkill / list.length).toFixed(1),
    averageWorkEfficiency: (totalWorkEfficiency / list.length).toFixed(1),
    averageCommunication: (totalCommunication / list.length).toFixed(1)
  };
  console.log('⭐ 从列表计算统计数据:', statistics);
}
```
